# Link URL here service menu

If you are missing the URL links in your folders that are present in other operating systems,
this service is going to fill the gap.

This service menu creates a *Desktop file* within the folder with the code that opens a given
URL using the *xdg-open* command, to launch the default browser, or a browser of your choice
among those supported.

Depending on the browser, it also takes into account some useful options to create the launcher,
like the private mode, the new window, and the profile.

## Features

These are the *Link URL here* service menu features:

- Licensed under the GPL-v3 license;
- Fully translated into 5 languages besides English
  (French, German, Italian, Portuguese, Spanish);
- Works from within the Dolphin, Konqueror, Krusader, and KFind results context menu;
- Support for KDE Plasma 6;
- Create a desktop file as a shortcut to open a given URL in one of the supported browsers;
- Support specific options for browser;
- Can use the default browser with the *xdg-open* command for a quick file creation;
- The supported browsers are Brave, Chromium, Google Chrome, Falkon, Firefox, and Vivaldi.

## Requirements

- KDE 4 or superior;
- KDialog;
- qtpaths6 for Plasma 6;
- xdg-open (default browser);
- jq (Brave, Chromium, Falkon, Google Chrome, Vivaldi);
- kwriteconfig/kreadconfig (Firefox).

## Install

To keep the latest stable version installed, updated, and easily removed from the same interface,
it is recommended to install this service menu through *Dolphin*:

- From Settings > Context Menu > Download new services;
- Search for *Link URL here*;
- Click on the *Install* button.

Also, the *Discover* application can install it:

- Under Application Addons > Servicemenu;
- Search for: *Link URL here*;
- Click on the *Install* button.

For debugging purpose follow one of the other methods mentioned on the [project page][installation].
